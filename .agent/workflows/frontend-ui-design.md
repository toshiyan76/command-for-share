# AIフロントエンドデザイン修正ワークフロー

このドキュメントは、AIが生成するフロントエンドコードの品質を向上させ、「AIっぽさ（AI Slop）」を排除し、プロフェッショナルで独自性のあるデザインを実現するためのワークフローを規定します。

## 1. 現状分析と「AIっぽさ」の監査
まず、既存のデザインまたは生成されたドラフトに対して、以下の「AIによる収束（Convergence）」の兆候がないか確認する。

*   **フォント**: Inter, Roboto, Open Sans, Segoe UI などのシステムデフォルトや無難なフォントが使われていないか？
*   **配色**: 均等に分布したパステルカラーや、白背景に紫のグラデーションといった「よくある」配色になっていないか？
*   **レイアウト**: 予測可能で退屈なグリッドや、コンテキストを無視したテンプレート的な配置になっていないか？
*   **動き**: 静的で、マイクロインタラクションやロード時のアニメーションが欠けていないか？

## 2. デザイン戦略の策定 (Steerability)
修正を行う前に、以下の3つの次元で明確なディレクションを決定する。

### A. タイポグラフィ戦略
*   **脱・無難**: Google Fonts等から、ブランドの個性に合った特徴的なフォントを選定する。
    *   *コード/テック系*: JetBrains Mono, Fira Code, Space Grotesk
    *   *エディトリアル/高級感*: Playfair Display, Crimson Pro
    *   *実用/堅牢*: IBM Plex Sans, Source Sans 3
*   **コントラスト**: フォントウェイト（100 vs 900）やサイズ（見出しは本文の3倍以上）で極端な差をつける。

### B. テーマと配色
*   **コンセプト定義**: 「RPG風」「サイバーパンク」「スイススタイル」「雑誌風」など、具体的な美的テーマを言語化する。
*   **カラーパレット**: CSS変数を定義し、ドミナントカラー（支配色）とシャープなアクセントカラーを決める。中間的な色は避ける。

### C. 深度と質感
*   **脱・単色**: 背景を単色（Solid Color）にせず、グラデーション、ノイズテクスチャ、幾何学パターンなどで奥行きを作る。

## 3. 実装プロンプトの適用 (Skills)
AIに対してコード生成/修正を指示する際は、以下の「Skill」としての制約を与える。

```markdown
<design_constraints>
1. Typography:
   - Inter/Robotoの使用を禁止。
   - [選定したフォント]を使用し、劇的なサイズコントラストをつけること。
2. Color & Theme:
   - [テーマ名]の美学に従うこと。
   - CSS変数を使用し、一貫性を保つこと。
3. Motion:
   - 静的な要素を禁止。
   - CSSアニメーション（@keyframes）でロード時のスタガー（時差）表示を実装すること。
   - ホバー時のマイクロインタラクションを必須とする。
4. Creativity:
   - "Think outside the box". 一般的なレイアウトパターンから逸脱し、驚きのあるデザインを選択すること。
</design_constraints>
```

## 4. 最終レビュー
生成されたデザインが以下を満たしているか確認する。
*   [ ] 一目で「AIが作った」と分からない独自性があるか？
*   [ ] ブランドの意図（高級感、親しみやすさ、堅牢さ等）が伝わるか？
*   [ ] ユーザーの操作に対して心地よい反応（モーション）があるか？
